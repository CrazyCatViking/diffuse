# Repository Guidance

When changing Diffuse behavior, update documentation in the same change.

- User-facing behavior changes belong in `README.md` or the relevant file under `docs/`.
- Architecture, process-boundary, persistence, JSON-RPC, or build-wiring changes belong in `docs/architecture.md`.
- Review data format changes belong in `docs/review-spec-v1.md`.
- LSP behavior or configuration changes belong in `docs/lsp.md`.
- Keep documentation links relative so they render correctly in GitHub.
